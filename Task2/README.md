# Задание 2. Разработка и заполнение проверочного листа для бизнес-систем


### Проблемы (выдержки из технического задания)

 - Нарушен контроль данных
 - Есть несколько точек регистрации нового клиента, координации между этими точками нет.
 - Внутренние потоки данных между IT-продуктами контролируются не системно
 - Ограничения по передаваемым данным в основном обусловлены бизнес-процессами между взаимодействующими IT-продуктами.
 - Со стороны систем обработки данных нет ограничений на доступ к данным.
 
 - API управляющих компаний без единых политик по безопасности и без системного контроля за передаваемыми данными
 - явно предоставляют персональные данные
 - внешним пользователям выполнять действия и получать данные, к которым у них не должно быть доступа


### Вектор для анализа

На основе проблем выше формирую вектор для анализа объектов безопасности:

 - Вопросы про разграничение контуров, сегментация. Дальнейшие вопросы по обеспечению безопасности контуров.
 - Вопросы по контролю доступа к сервисам и данным внутри каждого контура - RBAC/ABAC.


### Планы на развитие (выдержки из технического задания)

 - активно развивать сервисы для покупателей и жильцов. В том числе — за счёт партнёрских сервисов
 - клиентов: они видят в приложении ошибки, когда заходят в личный кабинет. Там иногда могут отображаться данные другого клиента — например, другое Ф. И. О
 - у конкурента произошла утечка данных
 - нет уверенности в том, что все текущие бизнес-процессы компании соответствуют законодательным нормам.

### Вектор для анализа

На основе планов развития выше формирую вектор для анализа объектов безопасности:

 - Вопросы классификации, хранения, доступа и передачи данных.
 - Вопросы про разграничение контуров, сегментации.
